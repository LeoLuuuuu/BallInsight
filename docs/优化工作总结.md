# BallInsight 系统优化工作总结

> 基于信息服务与信息检索课程项目，对 BallInsight 足球智能检索系统进行全方位优化。

## 1. 人工标注评价数据集

**文件**: `backend/relevance_judgments.json`

构建了包含 15 条查询的人工标注相关度评价数据集，涵盖球员、球队、赛事、昵称和边界场景：

- **标注方式**: 人工审查每条查询的返回结果标题和摘要，逐条赋予 0/1/2 分（不相关/部分相关/高度相关）
- **集成方式**: 修改 `search_engine.py`，优先使用人工标注，未标注结果回退到 `auto_relevance_grade` 自动评分
- **特殊处理**: 对 "C罗" 等已知返回 0 相关结果的查询，空标注集表示"已确认无相关文档"，所有结果强制标记为 0 分

**评价体系关键发现**:
- "C罗" 查询暴露了 Whoosh 混合中英文字符分词缺陷，所有 9 条返回结果均为误匹配（含"罗"字的无关文章），正确识别 P@10=0
- Baseline（BM25）和 Ours（BM25 × TimeDecay × Sentiment）在不同查询上各有优劣，体现了评价的客观性

## 3. 三模型 IR 对比评估

**文件**: `backend/search_engine.py`、`frontend/src/components/ABTestPanel.jsx`

新增 **TF-IDF 基线模型**，将原 A/B 双栏对比升级为三模型并列评估：

| 模型 | 算法 | 说明 |
|------|------|------|
| TF-IDF | 余弦相似度 | 经典词频-逆文档频率基线 |
| BM25 | Whoosh 原生评分 | 现代概率检索基线 |
| Ours | BM25 × TimeDecay × Sentiment | 融合时效衰减与情感权重 |

**实现细节**:
- 使用 scikit-learn `TfidfVectorizer` + jieba 分词构建 TF-IDF 索引，支持 unigram + bigram
- 前端 AB 测试面板升级为三列布局（黄/灰/绿配色区分）
- 指标图表新增 TF-IDF 柱状条
- 显示人工标注覆盖率提示

## 4. MMR 搜索结果多样性重排

**文件**: `backend/search_engine.py`

实现了 **Maximal Marginal Relevance** 算法：
- `λ=0.7` 平衡相关性与多样性
- 使用 Jaccard 相似度计算文档间内容重叠
- 通过 `?mmr=true` API 参数启用

## 5. 拼写纠错与查询建议

**文件**: `backend/main.py`、`frontend/src/App.jsx`

- **后端**: 新增 `/api/spell/check` 端点，基于 Levenshtein 编辑距离算法，结合球员名、球队名、同义词、足球术语构建的词典，返回相似度排序的前 5 条建议
- **前端**: 搜索返回 0 结果时自动调用纠错接口，展示可点击的建议标签，点击后自动重新搜索

```
"没西" → 梅西(0.5)、巴西(0.5)
"皇家马得里" → 皇家马德里(0.8)
"帽子西法" → 帽子戏法(0.75)
```
